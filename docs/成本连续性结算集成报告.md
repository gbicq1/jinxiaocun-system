#### 🎯 优化功能 2：后台队列管理

**实现位置：** `cost-settlement-service.ts`

**功能说明：**
- 所有结算任务通过队列顺序执行
- 避免并发冲突和数据库锁定
- 自动按顺序处理多个结算请求

**核心代码：**
```typescript
// 队列数据结构
private settlementQueue: Array<{
  year: number
  month: number
  lock: boolean
  resolve: (result: any) => void
  reject: (error: any) => void
}> = []

private isProcessingQueue = false

// 添加到队列
enqueueSettlement(year, month, lock) {
  return new Promise((resolve, reject) => {
    this.settlementQueue.push({ year, month, lock, resolve, reject })
    if (!this.isProcessingQueue) {
      this.processQueue()
    }
  })
}

// 处理队列
private async processQueue() {
  while (this.settlementQueue.length > 0) {
    const task = this.settlementQueue.shift()
    const result = this.settleMonth(task.year, task.month, task.lock)
    task.resolve(result)
    await new Promise(resolve => setTimeout(resolve, 100)) // 避免数据库锁定
  }
}
```

**使用场景：**
- 新增单据时自动触发结算
- 重新结算历史月份
- 手动触发月份结算

#### 🎯 优化功能 4：错误重试机制

**实现位置：** `cost-settlement-service.ts` 的 `processQueue` 方法

**功能说明：**
- 结算失败时自动重试，最多重试 3 次
- 使用指数退避策略（1 秒、2 秒、3 秒）
- 失败后重新加入队列等待重试

**核心逻辑：**
```typescript
if (retryCount < 3) {
  console.warn(`⚠️  任务失败，准备重试：${year}年${month}月 (第 ${retryCount + 1} 次)`)
  await new Promise(resolve => setTimeout(resolve, 1000 * (retryCount + 1)))
  task.reject(error)
  this.settlementQueue.unshift(task) // 重新加入队列
  retryCount++
  continue
} else {
  console.error(`❌ 任务失败，已达最大重试次数：${year}年${month}月`)
  task.reject(error)
}
```

**重试策略：**
- 第 1 次重试：等待 1 秒
- 第 2 次重试：等待 2 秒
- 第 3 次重试：等待 3 秒
- 超过 3 次：标记为失败

## 🔧 核心改进

### 1. 统一的成本结算接口

所有业务模块现在都使用统一的 `handleDocumentSave` 函数：

```typescript
import { handleDocumentSave, DocumentType } from '@/utils/cost-recalculation'

// 在保存单据后调用
await handleDocumentSave(
  DocumentType.PURCHASE_INBOUND,  // 单据类型

  formData.voucherDate             // 单据日期
)
```

### 2. 自动检测与触发

系统会自动检测：
- ✅ 新增当前月份单据 → 自动结算当前月份
- ✅ 新增历史月份单据 → 自动从该月份开始重算
- ✅ 修改已结算月份单据 → 自动从该月份开始重算
- ✅ 系统启动 → 自动补全未结算月份

### 3. 连续性保证

通过队列管理和自动重算机制，确保：
- ✅ 成本数据从第一笔业务开始连续计算
- ✅ 不同会计期间的成本价一致
- ✅ 上月期末 = 本月期初
- ✅ 并发操作不会导致数据冲突
