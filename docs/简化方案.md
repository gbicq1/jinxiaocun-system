# 简化方案：渲染进程成本结算

## 问题分析

成本结算服务在 Electron 主进程中运行，但需要访问：
1. localStorage（浏览器 API，主进程无法访问）
2. 数据库（主进程可以访问）

## 解决方案

**将成本结算逻辑移到渲染进程**：
- 保持当前代码基本不变
- 成本结算服务在浏览器环境中运行
- 可以访问 localStorage（业务数据）
- 可以通过 IPC 访问数据库（成本结算数据）

## 优点

1. ✅ 改动最小
2. ✅ 保持现有架构
3. ✅ 快速部署
4. ✅ 易于调试

## 缺点

1. ⚠️ 性能略差（但不是瓶颈）
2. ⚠️ 架构不够清晰（但可接受）

## 实施步骤

1. 恢复成本结算服务到渲染进程
2. 修改 IPC 接口，让前端调用数据库
3. 测试运行

## 后续优化

系统稳定后，可以：
1. 将所有业务数据迁移到数据库
2. 移除 localStorage 依赖
3. 将成本结算服务移回主进程
