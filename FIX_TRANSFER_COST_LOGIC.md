# 调拨单成本价获取逻辑修复

## 问题描述

库存调拨单自动获取的成本价与成本结算模块显示的加权平均单价不一致。

### 问题表现

- **调拨单获取的成本价**：再生稻 ¥31.45、苹果 ¥29.09
- **成本结算模块显示的加权平均单价**：再生稻 ¥22.15、苹果 ¥20.00
- **差异**：再生稻 ¥9.30、苹果 ¥9.09

## 根本原因

之前的实现逻辑过于复杂，试图从成本结算汇总数据中获取 `avgPrice`，但存在以下问题：

1. 依赖成本结算模块已结算的数据
2. 当期末库存为 0 时，计算逻辑会出错
3. 没有按照业务发生的实际顺序获取成本价

## 解决方案

### 新逻辑

**核心思想**：直接根据调拨单日期和调出仓库，查询该仓库在该日期的最后一笔业务的库存结余成本价。

**查询规则**：
1. 从调拨日期开始，往前查询该产品的所有业务单据
2. 找到最后一笔业务（离调拨日期最近的），使用该笔业务后的库存结余成本价
3. 如果调拨日期当天没有业务，则继续往前找
4. 如果整个月都没有业务，则使用上月结转的库存结余成本价

### 实现逻辑

```typescript
/**
 * 从成本结算模块获取指定日期的最终成本价
 * 逻辑：根据调拨日期和仓库，查询该仓库在该日期的最后一笔业务的库存结余成本价
 * 如果当天没有业务，则往前查询，直到查询到成本价
 * 如果整个月都没有业务，则使用上月结转的库存结余成本价
 */
export const getCostFromSettlement = (productId: number | undefined, warehouseId: number | undefined, transferDate: string): number => {
  // 1. 获取所有库存单据（入库、出库、调拨等）
  const allRecords = getAllInventoryRecords()
  
  // 2. 筛选出匹配产品和仓库的单据，按日期降序排序
  const filteredRecords = allRecords
    .filter(rec => 仓库匹配 && 产品匹配)
    .sort((a, b) => 日期降序)
  
  // 3. 从调拨日期开始，往前查找最后一笔业务
  for (const rec of filteredRecords) {
    if (recDate <= transferDate) {
      // 计算该笔业务后的库存结余成本价
      const stockInfo = calculateStockAfterTransaction(productId, warehouseId, recDate, allRecords)
      if (stockInfo.avgCost > 0) {
        return stockInfo.avgCost  // 找到并返回
      }
    }
  }
  
  // 4. 如果调拨日期前没有找到业务，递归获取上月最后一天的成本价
  const previousCost = getCostFromSettlement(productId, warehouseId, 上月最后一天)
  return previousCost
}
```

### 新增辅助函数

#### 1. getAllInventoryRecords()

获取所有类型的库存单据：
- 采购入库
- 销售出库
- 库存调拨
- 其他入库
- 其他出库

#### 2. calculateStockAfterTransaction()

计算指定日期某笔业务后的库存结余成本价：
- 按日期顺序遍历所有单据
- 根据单据类型（入库/出库/调拨）计算库存变化
- 返回库存结余的数量、总成本、平均成本

## 示例说明

### 示例 1：调拨日期有业务

```
调拨日期：2026-03-21
调出仓库：农服

业务记录：
- 2026-03-01：采购入库 10 个，单价 ¥10，总成本 ¥100
- 2026-03-10：销售出库 3 个
- 2026-03-15：采购入库 5 个，单价 ¥12，总成本 ¥60
- 2026-03-21：销售出库 2 个  ← 调拨日期

查询逻辑：
1. 找到 2026-03-21 的业务（销售出库 2 个）
2. 计算该业务后的库存结余：
   - 数量：10 - 3 + 5 - 2 = 10 个
   - 总成本：¥100 + ¥60 = ¥160
   - 平均成本：¥160 / 10 = ¥16.00

返回成本价：¥16.00
```

### 示例 2：调拨日期无业务，往前找

```
调拨日期：2026-03-25
调出仓库：农服

业务记录：
- 2026-03-01：采购入库 10 个，单价 ¥10
- 2026-03-15：采购入库 5 个，单价 ¥12
- 2026-03-21：销售出库 2 个

查询逻辑：
1. 2026-03-25 没有业务，往前找
2. 找到 2026-03-21 的销售出库
3. 计算 2026-03-21 后的库存结余成本价

返回成本价：¥16.00（同上）
```

### 示例 3：当月无业务，使用上月结转

```
调拨日期：2026-03-05
调出仓库：农服

3 月份业务记录：无
2 月份业务记录：
- 2026-02-28：采购入库 10 个，单价 ¥15

查询逻辑：
1. 3 月份没有业务
2. 递归获取 2 月份最后一天（2026-02-28）的成本价
3. 计算 2026-02-28 后的库存结余成本价

返回成本价：¥15.00
```

## 注意事项

### 1. 成本计算准确性

成本价是根据实际业务单据计算的，确保：
- 入库单的成本价准确
- 出库单的数量准确
- 调拨单的调出/调入仓库正确

### 2. 单据日期

所有单据必须有正确的日期，系统会按日期顺序计算库存结余。

### 3. 上月结转

如果当月没有业务，会递归查询上月最后一天的成本价，确保成本价的连续性。

### 4. 库存为 0 的情况

如果查询到的最后一笔业务后库存为 0，会继续往前找，直到找到库存大于 0 的业务。
