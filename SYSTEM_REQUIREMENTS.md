## 📊 数据库说明

### 数据库类型
- SQLite 3.x（单文件数据库）

### 数据库位置
```
C:\Users\<用户名>\AppData\Roaming\Electron\inventory.db
```

### 数据库大小
- 初始大小：~100KB
- 典型大小：10-50MB（1 万条业务数据）
- 最大支持：140TB（SQLite 理论限制）

### 备份建议
- 定期复制 `inventory.db` 文件到其他位置
- 建议使用 U 盘或云存储进行异地备份
- 系统未来将支持自动备份功能

## 🚀 性能优化建议

### 针对小型超市场景

1. **数据量控制**
   - 建议产品数量：< 10,000 个
   - 建议年业务单量：< 50,000 单
   - 定期归档历史数据（>3 年）

2. **硬件建议**
   - 使用 SSD 硬盘可显著提升性能
   - 8GB 内存可保证流畅运行
   - 避免同时运行多个大型程序

3. **维护建议**
   - 每月进行一次数据备份
   - 定期清理回收站
   - 避免数据库文件损坏

## 🔍 系统兼容性检查

### 检查 Windows 版本

**方法 1：使用运行命令**
```
win + R → 输入 "winver" → 查看版本
```

**方法 2：使用设置**
```
设置 → 系统 → 关于 → 查看 Windows 规格
```

**方法 3：使用命令行**
```powershell
systeminfo | findstr /B /C:"OS 名称" /C:"OS 版本"
```

### 检查系统架构

```powershell
# PowerShell 命令
Get-ComputerInfo | Select-Object OsArchitecture, WindowsVersion
```

### 检查可用资源

```powershell
# 检查内存
Get-CimInstance Win32_PhysicalMemory | Measure-Object -Property Capacity -Sum

# 检查硬盘空间
Get-Volume
```

## ⚠️ 已知限制

### 功能限制

1. **并发用户数**
   - 单用户设计（单机单用户）
   - 不支持多用户同时访问同一数据库

2. **网络功能**
   - 纯离线设计
   - 不支持云端同步（除非手动复制数据库）

3. **数据量限制**
   - 建议产品数 < 100,000
   - 建议年单量 < 100,000
   - 超过此规模建议升级服务器版本

### 技术限制

1. **打印机支持**
   - 支持标准 Windows 打印机
   - 小票打印机需要 Windows 驱动

2. **条码扫描枪**
   - 支持 USB 接口扫描枪（模拟键盘输入）
   - 支持蓝牙扫描枪（需配对）

3. **客户显示屏**
   - 需要额外配置
   - 支持串口/USB 显示屏
